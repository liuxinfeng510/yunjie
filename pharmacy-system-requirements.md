# 药房管理系统（YF）需求分析

## 一、项目概述

### 1.1 项目背景
开发一套符合GSP（药品经营质量管理规范）和三级等保要求的现代化药房管理系统，面向单体药店及小型连锁药店（10家以内），采用SaaS多租户架构，提供PC端+移动端全场景覆盖。

### 1.2 目标用户
- **药店经营者**：店长、老板，关注经营数据、库存周转、利润分析
- **药店店员**：收银员、导购，日常销售、入库、盘库操作
- **药剂师**：处方审核、用药指导、慢病管理
- **会员顾客**：微信小程序端查询、购药记录、缺药登记

### 1.3 技术选型
| 层级 | 技术栈 |
|------|--------|
| 后端 | Spring Boot 3.2 + Java 17 + MyBatis Plus |
| 前端 | Vue 3 + Vite + Element Plus |
| 移动端（店员） | UniApp + Vue 3 |
| 移动端（顾客） | 微信小程序 |
| 数据库 | MySQL 8.0（多租户模式） |
| 缓存 | Redis |
| AI能力 | 阿里云百炼 qwen-max |
| 部署 | 腾讯云/阿里云 SaaS |

## 二、核心功能模块

### 2.1 追溯码管理（GSP核心要求）

#### 功能要求
- **快捷扫码录入**：支持扫码枪、手机摄像头扫描药品追溯码
- **追溯码格式**：支持国家药监局20位追溯码、UDI码、医保追溯码
- **批量导入**：支持Excel批量导入追溯码数据
- **追溯查询**：一码追溯药品从入库到销售的全链路信息
- **追溯上报**：预留国家药品追溯平台接口

#### 数据结构
```
追溯码表(drug_trace_code)
├── trace_code        # 追溯码（唯一）
├── drug_id           # 药品ID
├── batch_no          # 批号
├── produce_date      # 生产日期
├── expire_date       # 有效期
├── supplier_id       # 供应商ID
├── purchase_order_id # 入库单ID
├── sale_order_id     # 销售单ID（已售）
├── status            # 状态：在库/已售/退货/报损
└── trace_time        # 追溯记录时间
```

### 2.2 效期管理（GSP核心要求）

#### 功能要求
- **效期预警**：可配置预警天数（30/60/90天），多级预警
- **近效期提醒**：每日自动推送近效期药品清单
- **效期商品促销**：自动标记近效期商品，支持设置促销价
- **过期处理流程**：过期药品自动锁定，需走报损流程
- **效期报表**：效期分布报表、近效期汇总报表

#### 业务规则
- 效期<30天：红色预警，禁止销售（可配置）
- 效期30-60天：橙色预警，提示促销
- 效期60-90天：黄色预警，关注
- 先进先出原则（FIFO）：系统自动推荐先出效期近的批次

#### 2.8.2 AI视觉自助结算（重点功能）

##### 功能场景
顾客将选购的药品放置在结算台上，系统通过摄像头自动识别所有药品及数量，实现快速结算。

##### 识别方式
| 药品类型 | 识别方式 | 技术方案 |
|----------|----------|----------|
| 有条码药品 | 条码/追溯码识别 | 多角度条码扫描 + OCR |
| 无码药品 | 图像识别 | AI视觉识别药品外观 |
| 中药饮片 | 名称+称重 | 电子秤+AI识别 |

##### 技术架构
```
AI视觉结算系统架构
├── 硬件层
│   ├── 高清摄像头（俯视角度，4K分辨率）
│   ├── 补光灯（确保光线均匀）
│   ├── 结算台（固定拍摄区域）
│   └── 显示屏（实时显示识别结果）
├── 识别层
│   ├── 条码识别引擎（ZXing/ZBar）
│   ├── OCR追溯码识别
│   ├── 药品图像识别模型（CNN/YOLO）
│   └── 数量检测（目标计数）
└── 业务层
    ├── 药品匹配（与药品库比对）
    ├── 价格查询
    ├── 库存扣减
    └── 订单生成
```

##### 识别流程
```
顾客放置药品 → 摄像头采集图像 → 多目标检测（识别每个药品）
    → 条码区域检测 → 有码：解析条码 / 无码：AI图像识别
    → 药品匹配 → 数量统计 → 显示结算清单
    → 顾客确认 → 支付 → 出库
```

##### 无码药品AI识别
- **训练数据**：采集店内所有药品的多角度图片
- **识别模型**：基于YOLOv8/ResNet的药品分类模型
- **置信度处理**：低置信度药品人工确认
- **增量学习**：新品种自动加入训练集

##### 多药品同时识别
- 支持同时识别10+个药品
- 自动区分相同药品的数量
- 重叠/遮挡药品提示调整位置

##### 数据结构
```
AI识别记录表(ai_vision_recognition)
├── session_id        # 结算会话ID
├── image_url         # 采集图片URL
├── recognition_time  # 识别耗时(ms)
├── items             # 识别结果JSON
│   ├── drug_id       # 药品ID
│   ├── drug_name     # 药品名称
│   ├── quantity      # 数量
│   ├── method        # 识别方式(barcode/ocr/vision)
│   └── confidence    # 置信度
├── manual_adjusted   # 是否人工调整
└── sale_order_id     # 关联销售单
```

#### 2.8.4 OCR智能入库（重点功能）

##### 功能场景
| 场景 | 说明 |
|------|------|
| 拍照入库 | 手机拍摄药品包装，自动识别信息 |
| 批量扫描 | 连续拍照多个药品，批量识别 |
| 发票识别 | 拍摄采购发票，自动匹配入库单 |
| 资质识别 | 供应商资质证书OCR识别 |

##### OCR识别字段
```
药品包装识别
├── 药品名称（通用名/商品名）
├── 规格
├── 生产批号
├── 生产日期
├── 有效期至
├── 生产企业
├── 批准文号/国药准字
├── 追溯码（条码区域）
└── 剂型
```

##### 技术方案
- **OCR引擎**：阿里云OCR / 百度OCR / 腾讯OCR
- **自定义模型**：针对药品包装训练专用模型
- **多角度识别**：支持倾斜、模糊图片增强处理
- **置信度阈值**：低置信度字段标记人工复核

##### 入库流程优化
```
传统流程：手工录入药品名 → 手工录入批号 → 手工录入效期 → ...
OCR流程：拍照 → 自动识别 → 人工确认/修正 → 一键入库
```

##### 数据结构
```
OCR识别记录表(ocr_recognition_log)
├── image_url         # 原始图片URL
├── recognition_result # 识别结果JSON
├── confidence_score  # 整体置信度
├── manual_corrected  # 是否人工修正
├── correction_fields # 修正字段记录
├── drug_id           # 匹配的药品ID

```

#### 2.11.4 智能电子秤集成（重点功能）

##### 功能场景
| 场景 | 说明 |
|------|------|
| 智能入库 | 放置中药 → 自动识别品种 → 自动读取重量 → 一键入库 |
| 智能出库 | 选择品种 → 放上电子秤 → 自动读取 → 确认出库 |
| 调配称重 | 处方调配时 → 显示应称量 → 实时显示已称量 → 自动累计 |
| 盘点称重 | 整斗称重 → 自动记录 → 对比系统库存 |

##### 硬件对接
```
智能电子秤系统架构
├── 硬件设备
│   ├── 高精度电子秤（精度0.1g，量程5kg）
│   ├── 摄像头（俯视角度，识别药品）
│   ├── 显示屏（显示药品信息、称重数据）
│   └── 通讯模块（串口RS232/USB/蓝牙）

│   ├── 重量数据实时读取

│   ├── 去皮操作
│   └── 清零操作
└── 软件层
    ├── 设备驱动（跨平台）
    ├── 数据解析
    ├── AI药品识别
    └── 业务逻辑
```

##### AI药品识别
中药饮片没有条码，通过摄像头+AI识别药品品种：
- **图像识别**：基于药品外观特征（颜色、形态、纹理）
- **特征提取**：训练中药饮片专用识别模型
- **辅助确认**：显示TOP3匹配结果供选择
- **自学习**：识别错误后人工修正，持续优化模型

##### 智能入库流程
```
传统流程：选择药品 → 手工输入重量 → 确认入库
智能流程：放置药品 → AI识别品种 → 电子秤读取重量 → 确认入库
```

##### 智能出库流程
```
选择出库品种 → 显示当前库存
    → 放置药品到秤上 → 实时显示重量
    → 确认出库重量 → 自动扣减库存
```

##### 调配辅助称重
```
加载处方 → 显示第一味药（药名+应称量+斗格位置）
    → 放置药品 → 实时显示已称量
    → 达到目标量提示 → 超量报警
    → 确认 → 自动跳转下一味药
    → 全部完成 → 累计总重校验
```

#### 2.11.5 中药调配

##### 称重智能校验

###### 校验规则
| 校验项 | 规则 | 处理方式 |
|--------|------|----------|
| 品种校验 | AI识别称上药品是否正确 | 错误报警，禁止继续 |
| 重量校验 | 实际重量 vs 应称重量 | 允许±5%误差，超出提示 |
| 累计校验 | 全部药品总重校验 | 完成后对比理论总重 |
| 顺序校验 | 特殊药品顺序（先煎后下等） | 提醒特殊处理 |

###### 品种校验流程
```
称上放置药品 → 摄像头采集图像 → AI识别药品品种
    → 与当前应调配药品比对
    → 匹配：显示重量，允许继续
    → 不匹配：报警"品种错误！当前应调配XXX，识别到YYY"
```

###### 重量校验流程
```
实时读取电子秤重量 → 与目标重量比对
    → <目标-5%：显示"不足"，继续添加
    → 目标±5%：显示"合格"，可确认
    → >目标+5%：显示"超量"，警告提示
    → >目标+20%：强制报警，需主管确认
```
