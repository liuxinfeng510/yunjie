# YF药房管理系统 - 开发规范

## 1. 商品搜索规范

### 1.1 统一搜索条件
所有涉及商品（药品）搜索的功能，必须支持以下4个搜索条件：

| 序号 | 搜索字段 | 数据库字段 | 说明 |
|------|----------|-----------|------|
| 1 | 商品名称 | generic_name, trade_name | 通用名、商品名 |
| 2 | 拼音 | pinyin, pinyin_short | 全拼、简拼 |
| 3 | 条码 | barcode | 商品条形码 |
| 4 | 批准文号 | approval_no | 药品批准文号 |

### 1.2 后端实现标准
```java
// 标准搜索条件实现
if (StringUtils.hasText(keyword)) {
    wrapper.and(w -> w
        .like(Drug::getGenericName, keyword)
        .or().like(Drug::getTradeName, keyword)
        .or().like(Drug::getPinyin, keyword)
        .or().like(Drug::getPinyinShort, keyword)
        .or().like(Drug::getBarcode, keyword)
        .or().like(Drug::getApprovalNo, keyword)
    );
}
```

### 1.3 前端实现标准
- 搜索框placeholder：`商品名称/拼音/条码/批准文号`
- 搜索框聚焦时禁用输入法（ime-mode: disabled）
- 支持键盘导航（↑↓选择，Enter添加）

### 1.4 适用范围
- POS收银药品搜索
- 库存管理药品查询
- 入库单药品选择
- 出库单药品选择
- GSP验收/养护药品选择
- 所有涉及药品选择的场景

## 2. 会员搜索规范

### 2.1 统一搜索条件
所有涉及会员信息搜索的功能，必须支持以下3个搜索条件：

| 序号 | 搜索字段 | 数据库字段 | 说明 |
|------|----------|-----------|------|
| 1 | 姓名 | name | 会员姓名 |
| 2 | 拼音 | pinyin, pinyin_short | 全拼、简拼 |
| 3 | 手机号码 | phone | 会员手机号 |

### 2.2 后端实现标准
```java
// 标准会员搜索条件实现
if (StringUtils.hasText(keyword)) {
    wrapper.and(w -> w
        .like(Member::getName, keyword)
        .or().like(Member::getPinyin, keyword)
        .or().like(Member::getPinyinShort, keyword)
        .or().like(Member::getPhone, keyword)
    );
}
```

### 2.3 前端实现标准
- 搜索框placeholder：`姓名/拼音/手机号`
- 支持模糊匹配

### 2.4 适用范围
- POS收银会员选择
- 会员管理列表查询
- 会员积分查询
- 用药提醒会员选择
- 所有涉及会员选择的场景

---

## 3. 术语规范

### 3.1 界面显示
- 统一使用"**商品名称**"，不使用"药品名称"
- 统一使用"**商品**"，不使用"药品"（特定场景除外）

### 3.2 代码命名
- 实体类/字段保持 `Drug`、`drugId` 等命名（不改动）
- 仅修改用户可见的界面文字
